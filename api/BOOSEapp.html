Namespace BOOSEapp

Classes

AppArray

This class represents a custom array used in the BOOSE interpreter. It can store either integer or real decimal values depending on how it is created. The class allows values to be added using the Poke method and returned using the Peek method. It also checks that the index is valid to prevent errors.

AppCanvas

This class represents the drawing area used in the BOOSE program. It is responsible for drawing shapes, lines, and text on the screen. The class controls the pen position, meaning where drawing starts, and also manages the pen colour and thickness. It provides different methods that allow the program to draw shapes such as circles and rectangles, draw lines between points, move the pen without drawing, and write text on the canvas. This class connects BOOSE commands to the visual output.

AppCircle

This class represents the circle command in the BOOSE system. It is used to draw a circle on the canvas using a given radius. The class takes one parameter, which is the size of the circle, and passes it to the canvas to perform the drawing. It also checks that the value is valid before drawing, ensuring that the radius is a positive number and not a decimal.

AppCommandFactory

This class is responsible for creating BOOSE command objects. It works as a factory, meaning it creates the correct command based on the command name given by the user. Each command name such as circle, rect, moveto is matched to a specific class that performs that action. If the command is not recognised, the base factory is used instead. This makes the system flexible and easy to extend with new commands.

AppIf

This class represents an If command in the BOOSE system. It is used to control whether a block of code should run or not. The condition itself is evaluated by the Parser, and this class stores the result of that condition. If the condition is true, the code inside the if block will run. If it is false, the block will be skipped. This class works together with the Parser to control program flow.

AppInt

This class represents an integer variable used in the BOOSE application. It stores a single whole number value that can be used in commands. The class allows the value to be set and retrieved, and it also provides a string version of the value when needed. This is useful when working with variables in BOOSE programs.

AppMoveTo

This class represents the MoveTo command in the BOOSE system. It is used to change the current drawing position on the canvas. The command takes two integer values, X and Y, which represent the new position of the pen. No drawing is done when this command is used. It only moves the pen to a new location, so the next drawing command starts from that position.

AppReal

This class represents a real decimal variable used in the BOOSE application. It stores a numeric value that can include decimal numbers. The class allows the value to be set and accessed and it can also return the value as text when needed. This is useful when working with decimal values in BOOSE programs.

AppRectangle

This class represents the rectangle command in the BOOSE system. It is used to draw a rectangle on the canvas using a given width and height. The command takes two integer values which define the size of the rectangle. The rectangle is drawn starting from the current pen position. The class also checks that the values are valid before drawing ensuring that width and height are positive integers.

AppWhile

This class represents a While command in the BOOSE system. It is used to repeat a block of code while a condition is true. The condition is evaluated by the Parser and this class works together with the Parser to control the loop. As long as the condition is true, the commands inside the while block will continue to run. When the condition becomes false, the loop stops. This class helps manage repetition in BOOSE programs.

AppWrite

This class represents the write command in the BOOSE system. It is used to display text on the canvas at the current pen position. The command takes one parameter which is the text to be shown. This allows messages or values to be displayed as part of the drawing.

ApplicationConfiguration

Bootstrap the application configuration.

Form1

This class represents the main user interface of the BOOSE application. It allows the user to write, run and view BOOSE programs. The form connects all main components of the system including the canvas, command factory, stored program and parser. It is responsible for taking user input, processing the BOOSE code and displaying the output on the screen.

Parser

This class is responsible for reading and processing BOOSE code. It acts as a custom parser that prepares the program before execution. The parser handles : variables (int and real), arrays (create, poke, peek), control structures (if, else, while) and mathematical expressions. It converts the user input into a final version of the program where all variables are replaced with actual values and only valid commands are passed to the BOOSE system. This class works as the main logic controller.

Program
VariableStore

This class stores and manages all variables used in the BOOSE program. It supports integer variables, real decimal variables and arrays. The class uses a Singleton pattern, meaning only one instance exists and is shared across the application. It allows variables to be created, updated, retrieved and cleared and also provides support for array operations such as poke and peek. This class acts as the central storage for all program data.
