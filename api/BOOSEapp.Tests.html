Namespace BOOSEapp.Tests

Classes

AppArrayTests

This class contains unit tests for the AppArray class. It checks that the array works correctly when it is created, when values are stored using Poke, and when values are retrieved using Peek. It also tests error handling by checking that invalid indexes throw exceptions. These tests help ensure that the AppArray class behaves correctly in different situations.

AppIfTests

This class contains unit tests for the AppIf command. It checks that the command can be created and executed safely. Since the main logic of the If statement is handled by the Parser, these tests focus on ensuring that the AppIf class does not crash and behaves correctly when its methods are called. These tests help confirm that the class works correctly within the overall BOOSE system.

AppIntTests

This class contains unit tests for the AppInt class. It checks that integer variables are created correctly, that values are stored properly and that they can be converted to a string when needed. These tests help ensure that the AppInt class behaves correctly in different situations.

AppRealTests

This class contains unit tests for the AppReal class. It checks that real (decimal) variables are created correctly, that values are stored and updated properly and that they can be converted to a string. These tests help ensure that the AppReal class behaves correctly when working with decimal values.

AppWhileTests

This class contains unit tests for while loop behaviour in the BOOSE interpreter. It checks that while blocks are handled correctly by the Parser especially when the condition is false. The tests focus on ensuring that while blocks are skipped when the condition is false, nested while loops are handled correctly, empty while blocks do not cause errors. These tests help confirm that loop control works correctly in the system.
